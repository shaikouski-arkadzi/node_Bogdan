# Последовательность работы

**Фазы:**

1. **Timers**  
   На этом этапе выполняются колбэки, запланированные через `setTimeout` и `setInterval`, если их время истекло.

2. **Pending Callbacks**  
   Здесь выполняются отложенные I/O колбэки, которые были перенесены на следующую итерацию цикла.

3. **Idle, Prepare**  
   Внутренняя фаза Node.js. Обычно не используется напрямую разработчиками.

4. **Poll**  
   Основная и самая важная фаза:
   - Получение новых I/O событий (например, чтение файлов, сетевые запросы)
   - Выполнение соответствующих колбэков
   - Если очередь пуста, возможен переход к следующей фазе

5. **Check**  
   Выполняются колбэки, запланированные через `setImmediate`.

6. **Close Callbacks**  
   Обрабатываются события закрытия, например `socket.on('close')`.

## process.nextTick и Promise

После выполнения **любого колбэка** (в любой фазе):

1. Сначала выполняется очередь `process.nextTick`
2. Затем выполняется очередь `Promise` (`.then`, `.catch`, `.finally`)

И только потом Event Loop переходит к следующей фазе.

## setImmediate

`setImmediate` вызывает колбэк функцию на текущей или на следующей итерации цикла событий.

`setImmediate(()=>console.log('immediate cb'));`
